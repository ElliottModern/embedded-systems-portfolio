Title

Project 02 - STM32 Timer Interrupt LED Blink

Goal

Blink the onboard LD2 LED on the STM32 NUCLEO-F446RE using a hardware timer interrupt instead of a software delay loop.

Hardware

STM32 NUCLEO-F446RE
onboard LD2 LED (PA5)

Key concepts used

hardware timers
timer prescaler configuration
auto-reload register (ARR)
timer update interrupts
NVIC interrupt controller
interrupt service routines (ISR)

Timer configuration

Timer used
TIM2

Clock source
84 MHz

Prescaler
8399

Timer frequency
84 MHz / (8399 + 1) = 10 kHz

Timer tick period
100 µs

Auto-reload value
4999

Interrupt interval
5000 ticks × 100 µs = 500 ms

LED blink cycle
ON 500 ms
OFF 500 ms

Core code

Timer initialization

static void TIM2_Init(void)
{
    RCC->APB1ENR |= (1 << 0);     // Enable TIM2 clock

    TIM2->PSC = 8399;             // Prescaler
    TIM2->ARR = 4999;             // Auto reload

    TIM2->DIER |= (1 << 0);       // Enable update interrupt

    NVIC_EnableIRQ(TIM2_IRQn);    // Enable interrupt in NVIC

    TIM2->CR1 |= (1 << 0);        // Start timer
}

Interrupt service routine

void TIM2_IRQHandler(void)
{
    if (TIM2->SR & (1 << 0))
    {
        TIM2->SR &= ~(1 << 0);    // Clear interrupt flag
        GPIOA->ODR ^= (1 << 5);   // Toggle LED
    }
}

Main loop

while (1)
{
    // Main loop idle
}

Notes

The LED timing is now controlled by a hardware timer instead of CPU delay loops.
The CPU remains free while the timer runs independently.
When the timer overflows, an interrupt is generated and the ISR toggles the LED.
